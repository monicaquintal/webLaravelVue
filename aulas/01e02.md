2. Instalar o PHP (Linux)

- utilizar um recurso do Sistema Operacional nativo (apt-get) para trazer os pacotes, instalar e gerenciar.
- no terminal:

~~~
sudo apt-get update
sudo apt -y install php7.4
php --version
~~~

3. [Instalar e configurar o Composer](https://getcomposer.org/)

- gerenciador de dependências do PHP.

~~~
php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
php -r "if (hash_file('sha384', 'composer-setup.php') === 'c8b085408188070d5f52bcfe4ecfbee5f727afa458b2573b8eaaf77b3419b0bf2768dc67c86944da1544f06fa544fd47') { echo 'Installer verified'.PHP_EOL; } else { echo 'Installer corrupt'.PHP_EOL; unlink('composer-setup.php'); exit(1); }"
php composer-setup.php
php -r "unlink('composer-setup.php');"
php composer.phar --version
~~~
git push -u origin main
- para tornar o Composer global (caso queira - eu não fiz)
    - movimentar esse script para o diretório `/usr/local/bin`.
    - todos os binários contidos dentro desse diretório são reconhecidos automaticamente em qualquer local.

~~~
cd Documentos
sudo mv composer.phar /usr/local/bin/composer
ls -la
~~~

4. Iniciando um projeto com [Laravel](https://laravel.com/)

- pra iniciar projetos Laravel há 2 métodos distintos.
    - via Laravel Installer.
    - via Composer Create-Project.

a) via Composer:
~~~
composer create-project --prefer-dist laravel/laravel blog
~~~ 

- não utilizar no nome do projeto caracteres especiais ou espaço.
- para verificar se a aplicação está funcionando: acessar o diretório do projeto e a pasta public, e utilizar a instrução `php -S localhost:8000`.

b) via Laravel Installer

```
via global
composer global require laravel/installer
laravel new example-app
cd example-app
npm install && npm run build
composer run dev

local
composer create-project laravel/laravel projeto_laravel
cd projeto_laravel
npm install
npm run build
composer run dev
```
