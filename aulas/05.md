<h2 id="topico1">1. Encaminhando parâmetros da rota para o controlador</h2>

- em `web.php`, excluir a rota2 e o método route::redirect, manter o fallback, transformar a \rota1 em \teste e mudar a function() para TesteController.

~~~php
Route::get('/teste', 'TesteController@teste') -> name ('teste');

Route::fallback(function(){
    echo 'A rota acessada não existe! <a href= "'.route('site.index').'">Clique aqui</a> para ir para a página inicial :)';
});
~~~

- no terminal, utilizar o comando `php artisan make:controller TesteController`. Nesse Controlador de teste, implementar:

~~~php
<?php
namespace App\Http\Controllers;
use Illuminate\Http\Request;
class TesteController extends Controller
{
    public function teste(int $p1, int $p2){
        echo "A soma de $p1 + $p2 é: ".($p1+$p2);
    }
}
~~~

- não é necessário definir as variáveis que representam os parâmetros do método com o mesmo nome dos parâmetros recebidos na rota; o importante é a sequência, ou seja, a ordem de recebimento dos parâmetros.

<h2 id="#topico2">2. Encaminhando parâmetros do controlador para visualização</h2>

- para encaminhar parâmetros do Controlador para uma View, há algumas opções: `Array associativo`, `método Compact()` e `método With()`.
- no diretório resources > view > site, criar um novo arquivo chamado `teste.blade.php` e, em `TesteController.php`:

~~~php
class TesteController extends Controller
{
    public function teste(int $p1, int $p2){
        // echo "A soma de $p1 + $p2 é: ".($p1+$p2);
        return view('site.teste');
    }
}
~~~

### a) Array associativo

- construir um array utilizando os colchetes, e fazendo uma associação desses valores com chaves (índices), que serão recuperadas na View.
- em `TesteController.php`:

~~~php
<?php
class TesteController extends Controller
{
    public function teste(int $p1, int $p2){
        // echo "A soma de $p1 + $p2 é: ".($p1+$p2);
        return view('site.teste', ['x' => $p1, 'y' => $p2]);
    }
}
~~~

- e em `teste.blade.php`, o índice x se transforma na variável $x no contexto da view.

~~~php
P1 = {{ $x }}
<br>
P2 = {{ $y }}
~~~

- geralmente, no índice associativo, usamos o nome da própria variável; portanto, em `TesteController.php`:

~~~php
class TesteController extends Controller
{
    public function teste(int $p1, int $p2){
        // echo "A soma de $p1 + $p2 é: ".($p1+$p2);
        return view('site.teste', ['p1' => $p1, 'p2' => $p2]);
    }
}
~~~

- também ajustar em `teste.blade.php`:

~~~php
P1 = {{ $p1 }}
<br>
P2 = {{ $p2 }}
~~~

### b) Método Compact()

- nativo do próprio php.
- nesse caso, não passamos os valores como variáveis - passamos como strings, sem utilizar o $.
- em `TesteController.php`:

~~~php
return view('site.teste', compact('p1', 'p2'));
~~~

### c) Método With()

- nesse caso, passamos dois parâmetros: o nome da variável do lado da view e seu respectivo valor.

~~~php
return view('site.teste')->with('xyz', $p1)->with('abc', $p2);
~~~

<h2 id="#topico3">3. Sintaxe Blade</h2>

- o Blade é o motor de renderização de Views do Laravel.
- permite otimizar o tempo de desenvolvimento do frontend da aplicação.
- através dele temos acesso a uma sintaxe de programação php muito mais enxuta e clara, facilitando a criação de páginas web dinâmicas.
- além disso, o blade possui um poderoso recurso de herança de templates, que possibilita que views estendam, modifiquem ou incluam outras views.
- diferente de outros motores de renderização de views, o blade não impede o uso da sintaxe pura do php.

<h2 id="#topico4">4. Blade - Incluindo comentários e blocos PHP puros</h2>

- implementaremos o FornecedorController para a rota \fornecedores; nesse controlador, criaremos um método chamado index(), que apontará para uma nova view app.fornecedor.index.
- criar o controlador FornecedoresController através do comando `$ php artisan make:controller FornecedoresController`.

~~~php
<?php
namespace App\Http\Controllers;
use Illuminate\Http\Request;
class FornecedoresController extends Controller
{
    public function index() {
        return view('app.fornecedor.index');
    }
}
~~~

- no diretório app, criar um novo diretório chamado `fornecedores`, e dentro dele um arquivo `index`.
- em `web.php`:

~~~php
Route::get('/fornecedores', [\App\Http\Controllers\FornecedoresController::class, 'index']) -> name('app.fornecedores');
~~~

### Para criar comentários:

- utilizar chaves 2x e traço 2x, como no exemplo em `index.blade.php`:

~~~php
{{-- Comentário no Blade --}}
~~~

- esse comentário não aparecerá nem ao Inspecionar a página.

### Para incluir códigos PHP puros:

- iniciar o trecho que conterá esse código com `@php` e conclui-lo com `@endphp`.
- dessa forma, quando a view for processada, o blade identificará que trata-se de código em php puro, como no exemplo:

~~~php
{{-- Comentário no Blade --}}

@php
    // comentário com sintaxe de php puro!
    /* mais um comentário! */
@endphp
~~~

### Observação importante:

- `{{ }}` funciona exatamente como a tag curta de impressão do php `<?= =>`, como no exemplo:

~~~php
{{ 'Texto de teste' }}
<?= "Texto de teste" ?>

@php
    // comentário com sintaxe de php puro!
    echo 'Texto de teste';
@endphp
~~~

<h2 id="#topico6">6. Blade - @if/@else</h2>

- em `FornecedoresController.php`:

~~~php
class FornecedoresController extends Controller
{
    public function index() {
        $fornecedores = ['Fornecedor1'];
        return view('app.fornecedor.index', compact('fornecedores'));
    }
}
~~~

- no arquivo `index.blade.php`, não conseguimos utilizar a sintaxe "{{}}", pois $fornecedores trata-se de um array, e esse método não suporta a impressão de uma variável do tipo string. Portanto, para importar a variável do Controller, utilizaremos:

~~~php
@dd($fornecedores)
~~~

- **DETALHES IMPORTANTES**: 
    - na sintaxe do Blade não utilizamos o ";" para encerrar os comandos!
    - utilizamos o arroba @ para indicar comandos do que precisa ser interpretado pelo motor de renderização do blade.

- para utilizar os comandos if, elseif e else no blade: 

~~~php
@if(count($fornecedores) > 0 && count($fornecedores) < 10)
    <h3>Existem alguns fornecedores cadastrados.</h3>
@elseif(count($fornecedores) >= 10)
    <h3>Existem vários fornecedores cadastrados.</h3>
@else
    <h3>Ainda não existem fornecedores cadastrados.</h3>
@endif
~~~

<h2 id="#topico7">7. Blade - @unless</h2>

- é um operador de decisão; é uma inversão do operador if!
    - enquanto o if avalia o retorno positivo de uma condição, o unless faz o inverso.
- o unless não possui uma correspondência direta com um comando da linguagem de programação php - ele funciona mais como atalho.

~~~php
if (!<condicao>) {}
// if executa a condição se o retorno for true

{{ @unless }} 
// executa se o retorno for false
~~~

- para testar a utização do `unless`, em `FornecedoresController.php`:

~~~php
class FornecedoresController extends Controller
{
    public function index() {
        $fornecedores = [
            0 => ['nome' => 'Fornecedor1', 'status' => 'N']
            ];
        return view('app.fornecedor.index', compact('fornecedores'));
    }
}
~~~

- e em `index.blade.php`:

~~~php
Fornecedor: {{ $fornecedores[0]['nome'] }}
Status: {{ $fornecedores[0]['status'] }}
~~~

- lembrando que o comando `@dd($fornecedores)` exibe os valores que estão sendo importados; porém deve ser retirado antes de executar os comandos acima.
- aplicando if e unless:

~~~php
@if($fornecedores[0]['status']  == 'N')
    Fornecedor Inativo
@endif

// ou

@unless(($fornecedores[0]['status']  == 'S'))
    Fornecedor Inativo
@endunless
~~~

<h2 id="#topico8">8. Blade - @isset</h2>

- o `operador @isset` é utilizado para saber se variáveis(incluindo índices de arrays) estão ou não definidas - se existem ou não.
- na sintaxe blade, esse operador pode ser utilizado como um atalho, já que no php o isset é um método, utilizado dentro dos blocos de decisão ifelse.
- exemplo:

~~~php
@php
    /*
    if (isset($variavel)) {}
        retorna true se a variável estiver definida
    */
@endphp

@isset($fornecedores)
    Fornecedor: {{ $fornecedores[0]['nome'] }}
    <br>
    Status: {{ $fornecedores[0]['status'] }}
    <br>
@endisset
~~~

- agora, para fins de teste, será criado um novo índice em `FornecedoresController.php`, de número 1, porém sem fornecer o parâmetro cnpj:

~~~php
class FornecedoresController extends Controller
{
    public function index() {
        $fornecedores = [
            0 => [
                'nome' => 'Fornecedor1', 
                'status' => 'N', 
                'cnpj' => '00.000.000/0000-00'],
            1 => [
                'nome' => 'Fornecedor2', 
                'status' => 'S']   
            ];
        return view('app.fornecedor.index',);
    }
}
~~~

- em `index.blade.php`:

~~~php
@isset($fornecedores)
    Fornecedor: {{ $fornecedores[1]['nome'] }}
    <br>
    Status: {{ $fornecedores[1]['status'] }}
    <br>
    @isset($fornecedores[1]['cnpj'])
        CNPJ: {{ $fornecedores[1]['cnpj'] }}
    @endisset
@endisset
~~~

<h2 id="#topico9">9. Blade - @empty</h2>

- o `operador empty` serve para saber se uma variável ou índice de array está ou não vazio.
    - para o php, **uma variável está vazia se**: 
        - '' - string vazia
        - 0 - inteiro igual a zero
        - 0.0 - ponto flutuante zero
        - '0' - string contendo o caractere zero
        - valor null
        - valor false
        - array() - se for um array vazio
        - se a variável estiver apenas declarada (existe mas não possui atribuição de nenhum valor)
- na sintaxe blade, pode ser usado como um atalho, já que no php esse operador é um  método utilizado dentro dos blocos de decisão if/else.

- em `FornecedoresController.php`:

~~~php
class FornecedoresController extends Controller
{
    public function index() {
        $fornecedores = [
            0 => [
                'nome' => 'Fornecedor1', 
                'status' => 'N', 
                'cnpj' => ''],
            1 => [
                'nome' => 'Fornecedor2', 
                'status' => 'S']   
            ];
        return view('app.fornecedor.index', compact('fornecedores'));;
    }
}
~~~

- em `index.blade.php`:

~~~php
@php
    /*
    if (empty($variavel)) {}
        retorna true se a variável estiver vazia
    */
@endphp

@isset($fornecedores)
    Fornecedor: {{ $fornecedores[0]['nome'] }}
    <br>
    Status: {{ $fornecedores[0]['status'] }}
    <br>
    @isset($fornecedores[0]['cnpj'])
        CNPJ: {{ $fornecedores[0]['cnpj'] }}
        @empty($fornecedores[0]['cnpj'])
        Vazio!
        @endempty
    @endisset
@endisset

/*
Nesse caso, vai retornar:
    Fornecedor: Fornecedor1
    Status: N
    CNPJ: Vazio!
*/
~~~

<h2 id="#topico10">10. Extra - Operador condicional ternário</h2>

- recurso utilizado para minimizar a quantidade de código que escrevemos em nossos scripts.
- é um recurso nativo do php; não depende do framework laravel.
    - é um atalho do comando condicional if (escrito em uma linha, através da definição de 3 operandos que, apesar de não ser indicado, podem ser encadeados).
    - segue o modelo `condicao ? se verdadeiro : se falso;`.
        - para fazer encadeamento (mesmo que não seja o mais indicado), seguiria o padrão `condicao ? se verdadeiro : (condicao ? se verdadeiro : se falso);`.

- em `FornecedoresController.php`:

~~~php
class FornecedoresController extends Controller
{
    public function index() {
        $fornecedores = [
            0 => [
                'nome' => 'Fornecedor1', 
                'status' => 'N', 
                'cnpj' => '000'],
            1 => [
                'nome' => 'Fornecedor2', 
                'status' => 'S']   
            ];

        $msg = isset($fornecedores[0]['cnpj']) ? 'CNPJ informado' : 'CNPJ não informado';
        echo $msg;

        return view('app.fornecedor.index', compact('fornecedores'));
    }
}
~~~

- os operadores ternários são muito utilizados no processo de atribuição de valor para uma variável, em que faremos essa atribuição dependendo de alguma condição. Exemplo:

~~~php
$msg = isset($fornecedores[0]['cnpj']) ? 'CNPJ informado' : 'CNPJ não informado';
echo $msg;
~~~

<h2 id="#topico11">11. Blade - Operador condicional de valor default (??)</h2>

- funciona como um atalho para os operadores condicionais ternários.
- em `index.blade.php`:

~~~php
@isset($fornecedores)
    Fornecedor: {{ $fornecedores[0]['nome'] }}
    <br>
    Status: {{ $fornecedores[0]['status'] }}
    <br>
    CNPJ: {{ $fornecedores[0]['cnpj'] ?? 'Dado não foi preenchido' }}
    <!--
    SE
        - $variavel testada não estiver definida (isset)
            ou
        - $variavel testada possuir o valor null
    O valor default será usado no lugar da variável em questão!
    -->
@endisset
~~~

<h2 id="#topico12">12. Blade - @switch/case</h2>

- funciona de forma análoga ao operador if; com base em uma comparação, podemos usar o switch para determinar qual(is) bloco(s) de código deve(m) ser executado(s).
- em `FornecedoresController.php`:

~~~php
class FornecedoresController extends Controller
{
    public function index() {
        $fornecedores = [
            0 => [
                'nome' => 'Fornecedor1', 
                'status' => 'N', 
                'cnpj' => '0',
                'ddd' => '11',
                'telefone' => '0000-0000'],
            1 => [
                'nome' => 'Fornecedor2', 
                'status' => 'S',
                'cnpj' => null,
                'ddd' => '85',
                'telefone' => '0000-0000'],
            2 => [
            'nome' => 'Fornecedor3', 
            'status' => 'S',
            'cnpj' => null,
            'ddd' => '32',
            'telefone' => '0000-0000']
            ];

        return view('app.fornecedor.index', compact('fornecedores'));
    }
}
~~~

- em `index.blade.php`:

~~~php
@isset($fornecedores)
    Fornecedor: {{ $fornecedores[0]['nome'] }}
    <br>
    Status: {{ $fornecedores[0]['status'] }}
    <br>
    CNPJ: {{ $fornecedores[0]['cnpj'] ?? 'Dado não foi preenchido' }}
   <br>
    Telefone: {{ $fornecedores[0]['ddd'] ?? '' }} {{ $fornecedores[0]['telefone'] ?? '' }}
   @switch($fornecedores[0]['ddd'])
        @case('11')
            São Paulo - SP
            @break
        @case('32')
            Juiz de Fora - MG
            @break
        @case('85')
            Fortaleza - CE
            @break
        @default {{-- executada caso nenhum case satisfaça a condição --}}
            Estado não identificado
   @endswitch
@endisset
~~~

<h2 id="#topico13">13. Blade - @for</h2>

- o operador for é análogo ao comando for nativo do php; através dele podemos implementar estruturas de repetição nos scripts.
- exemplo da sintaxe:

~~~php
@for($i = 0; $i <= 10; $i++)
    {{ $i }} <br>
@endfor
~~~

- em `index.blade.php`:

~~~php
@isset($fornecedores)

    @for($i = 0; isset($fornecedores[$i]); $i++)
        Fornecedor: {{ $fornecedores[$i]['nome'] }}
        <br>
        Status: {{ $fornecedores[$i]['status'] }}
        <br>
        CNPJ: {{ $fornecedores[$i]['cnpj'] ?? 'Dado não foi preenchido' }}
        <br>
        Telefone: {{ $fornecedores[$i]['ddd'] ?? '' }} {{ $fornecedores[$i]['telefone'] ?? '' }}
        <hr>
    @endfor

@endisset
~~~

<h2 id="#topico14">14. Blade - @while</h2>

- é análogo ao comando while nativo do php.
- através dele, podemos implementar estruturas de repetição em nossos códigos.
- em `index.blade.php`:

~~~php
@isset($fornecedores)

    @php $i = 0; @endphp
    @while(isset($fornecedores[$i]))
        Fornecedor: {{ $fornecedores[$i]['nome'] }}
        <br>
        Status: {{ $fornecedores[$i]['status'] }}
        <br>
        CNPJ: {{ $fornecedores[$i]['cnpj'] ?? 'Dado não foi preenchido' }}
        <br>
        Telefone: {{ $fornecedores[$i]['ddd'] ?? '' }} {{ $fornecedores[$i]['telefone'] ?? '' }}
        <hr>
        @php $i++; @endphp
    @endwhile

@endisset
~~~

<h2 id="#topico15">15. Blade - @foreach</h2>

- também possui um equivalente nativo na linguagem de programação php.
- em `index.blade.php`:

~~~php
@foreach($fornecedores as $indice => $fornecedor)
    Fornecedor: {{ $fornecedor['nome'] }}
    <br>
    Status: {{ $fornecedor['status'] }}
    <br>
    CNPJ: {{ $fornecedor['cnpj'] ?? 'Dado não foi preenchido' }}
    <br>
    Telefone: {{ $fornecedor['ddd'] ?? '' }} {{ $fornecedor['telefone'] ?? '' }}
    <hr>
@endforeach
~~~

<h2 id="#topico16">16. Blade - @forelse</h2>

- é um a sintaxe do blade que não possui uma equivalência direta com algumn comando nativo do php.
- funciona como uma combinação do foreach e um comando condicional para verificar se o array está ou não vazio - ou seja, se houverem itens a serem percorridos no array, o bloco de repetição será executado, senão o fluxo será desviado.
- exemplo, em `index.html/php`:

~~~php
@isset($fornecedores)
    @forelse($fornecedores as $indice => $fornecedor)
        Fornecedor: {{ $fornecedor['nome'] }}
        <br>
        Status: {{ $fornecedor['status'] }}
        <br>
        CNPJ: {{ $fornecedor['cnpj'] ?? 'Dado não foi preenchido' }}
        <br>
        Telefone: {{ $fornecedor['ddd'] ?? '' }} {{ $fornecedor['telefone'] ?? '' }}
        <hr>
    @empty
        Não existem fornecedores cadastrados.
    @endforelse
@endisset
~~~

<h2 id="#topico17">17. Blade - Escapando a tag (sintaxe) de impressão do Blade</h2>

- em algumas situações, pode ser necessário utilizar os caracteres "{{}}", mas não para imprimir strings ou valores de variáveis, mas sim para escrever e imprimir esses caracteres em si.
- para indicar para o interpretador que essa informação não deve ser processada, basta utilizar à esquerda o caractere @, como em `Fornecedor: @{{ $fornecedor['nome'] }}`.
- exemplo:

~~~php
@forelse($fornecedores as $indice => $fornecedor)
        Fornecedor: @{{ $fornecedor['nome'] }}
~~~

<h2 id="#topico18">18. Blade - Variável loop</h2>

- o `objeto loop` é criado automaticamente pelo blade quando estamos trabalhando com os **laços foreach e forelse**.
- esse objeto loop fornece informações importantes sobre o loop que está acontecendo.
- exemplo, testando o `atributo iteration` em `index.blade.php`:

~~~php
@forelse($fornecedores as $indice => $fornecedor)
    Iteração atual: {{ $loop -> iteration }} 
    // comando iteration mostra a iteração atual!
    <br>
    Fornecedor: {{ $fornecedor['nome'] }}
    <br>
    Status: {{ $fornecedor['status'] }}
    <br>
    CNPJ: {{ $fornecedor['cnpj'] ?? 'Dado não foi preenchido' }}
    <br>
    Telefone: {{ $fornecedor['ddd'] ?? '' }} {{ $fornecedor['telefone'] ?? '' }}
    <hr>
@empty
    Não existem fornecedores cadastrados.
@endforelse
~~~

- testando o `atributo first` em `index.blade.php`, que indica se a iteração em questão é o primeiro índice do array:

~~~php
@forelse($fornecedores as $indice => $fornecedor)
    Iteração atual: {{ $loop -> iteration }} 
    <br>
~~~

<h2 id="#topico20">20. O que são assets?</h2>

- o termo `assets` refere-se a tudo aquilo que é utilizado para complementar o conteúdo do frontend das aplicações web (mídias como imagens, sons, vídeos, fontes e até mesmo estilos css e scripts javascript).

<h2 id="#topico21">21. Adicionando assets as views (helper asset)</h2>

- inseridos os arquivos de imagem no projeto.
- as imagens estão incluídas de modo relativo; a forma mais apropriada de incluir assets nas views é através da utilização de um helper do laravel chamado `helper asset`, como no exemplo abaixo:

~~~php
<img src="{{ asset('') }}">
~~~

<h2 id="#topico22">22. Adicionando arquivos CSS externos às páginas web</h2>

- dentro do `diretório public`, criar um novo diretório chamado `css`, e um arquivo `estilo_basico.css`; inserir todo o conteúdo css neste novo arquivo e importar as informações em principal, sobre-nos e contato através da tag link:

~~~php
<link rel="stylesheet" href="{{ asset('css/estilo_basico.css') }}">
~~~

- lembrando que o asset aponta diretamente para o diretório public!

- template é um recurso que ajuda a ter mais produtividade e organização quando estiver trabalhando com Views.

- a paertir de um template podemos reaproveitar uma grande quantidade de código sem ter que ficar reescrevendo tudo que é comum entre as páginas web.
