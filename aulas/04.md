<h2 id="topico1">1. Enviando parâmetros</h2>

- no arquivo `web.php`, criar uma nova rota:

~~~php
Route::get('/contato', [\App\Http\Controllers\ContatoController::class,'contato']);
// nome, categoria, assunto, mensagem

Route::get('/contato/{nome}', function (string $xyz) {
    echo 'Estamos aqui! :) '.$xyz;
});
~~~

- `{nome}` é um parâmetro fornecido que direciona para a nova rota; o nome corresponde à variável $xyz, que definimos como uma string.
- adicionar os demais parâmetros:

~~~php
Route::get('/contato', [\App\Http\Controllers\ContatoController::class,'contato']);
// nome, categoria, assunto, mensagem

Route::get(
    '/contato/{nome}/{categoria}/{assunto}/{mensagem}', 
    function (string $nome, string $categoria, string $assunto, string $mensagem) {
        echo "Estamos aqui: $nome - $categoria - $assunto - $mensagem";
    }
);
~~~

<h2 id="topico2">2. Parâmetros opcionais e valores padrões</h2>

- para tornar os parâmetros opcionais, indicar, à direita do seu nome, o `caractere ?`, e na função de callback informar uma mensagem padrão que será encaminhada no lugar do parâmetro ausente, como:

~~~php
Route::get(
    '/contato/{nome}/{categoria}/{assunto}/{mensagem?}', 
    function (string $nome, string $categoria, string $assunto, string $mensagem = 'Mensagem não informada :(') {
        echo "Estamos aqui: $nome - $categoria - $assunto - $mensagem";
    }
);
~~~

- o laravel possui uma limitação em relação à definição de parâmetros opcionais: devemos **sempre passar os parâmetros opcionais da direita para a esquerda**, para que o laravel saiba sequenciar os parâmetros que está recebendo.

~~~php
Route::get(
    '/contato/{nome?}/{categoria?}/{assunto?}/{mensagem?}', 
    function (
        string $nome = 'Desconhecido', 
        string $categoria = 'Informação', 
        string $assunto = 'Contato', 
        string $mensagem = 'Mensagem não informada :(') {
        echo "Estamos aqui: $nome - $categoria - $assunto - $mensagem";
    }
);
~~~

<h2 id="topico3">3. Tratando parâmetros de rotas com expressões regulares</h2>

- esse tipo de tratativa é útil para **aplicar controles**, de modo que as rotas sejam processadas **apenas** se os parâmetros passados estiverem dentro das condições especificadas nas expressões regulares.
- em `web.php`:

~~~php
Route::get(
    '/contato/{nome}/{categoria_id}', 
    /* a categoria será um select com as possibilidades, 
    e na rota seria trafegado um id*/
    function (
        string $nome = 'Desconhecido', 
        int $categoria_id = 1 // 1 = 'Informação'
        ) {
        echo "Estamos aqui: $nome - $categoria_id";
    }
);
~~~

- aplicando um tratamento nos parâmetros regulares da rota, utilizando expressões regulares para indicar quais as condições que apontam que o parâmetro está apto a ser recebido, como:

~~~php
Route::get(
    '/contato/{nome}/{categoria_id}', 
    function (
        string $nome = 'Desconhecido', 
        int $categoria_id = 1 // 1 = 'Informação'
        ) {
        echo "Estamos aqui: $nome - $categoria_id";
    }
) -> where('categoria_id','[0-9]+');
/* caracteres devem estar entre 0 e 9, 
e ser pelo menos 1 caractere enviado */
~~~

- e, para garantir os parâmetros de $nome:

~~~php
Route::get(
    '/contato/{nome}/{categoria_id}', 
    function (
        string $nome = 'Desconhecido', 
        int $categoria_id = 1 // 1 = 'Informação'
        ) {
        echo "Estamos aqui: $nome - $categoria_id";
    }
) -> where('categoria_id','[0-9]+') -> where('nome','[A-Za-z]+');
~~~

<h2 id="topico4">4. Super Gestão - Criando o menu de navegação</h2>

- implementaremos um meno de nagevação nas views, permitindo que naveguemos entre as rotas da aplicação.
- acessar `resources > views > site`, e nos arquivos `principal.blade.php`, `sobre-nos.blade.php` e `contato.blade.php`, inserir a lista com os links:

~~~html
<ul>
    <li>
        <a href="/">Principal</a>
    </li>
    <li>
        <a href="/sobre-nos">Sobre nós</a>
    </li>
    <li>
        <a href="/contato">Contato</a>
    </li>
</ul>
~~~

- em `web.php`, retirar a rota inserida para teste na aula passada.

~~~php
<?php

use Illuminate\Support\Facades\Route;

Route::get('/', [\App\Http\Controllers\PrincipalController::class, 'principal']);

Route::get('/sobre-nos', [\App\Http\Controllers\SobreNosController::class, 'sobreNos']);

Route::get('/contato', [\App\Http\Controllers\ContatoController::class,'contato']);
~~~

- o php artisan possui um comando que permite listar as rotas criadas dentro do projeto: `php artisan route:list`.

~~~
GET|HEAD  / ................ PrincipalController@principal
GET|HEAD  contato .......... ContatoController@contato
GET|HEAD  sobre-nos ........ SobreNosController@sobreNos
~~~

<h2 id="topico5">5. Super Gestão - Implementando as rotas login, clientes, fornecedores e produtos</h2>

- criaremos 4 novas rotas: login, clientes, fornecedores e produtos.

~~~php
Route::get('/login', function() {return 'Login';});

Route::get('/clientes', function() {return 'Clientes';});

Route::get('/fornecedores', function() {return 'Fornecedores';});

Route::get('/produtos', function() {return 'Produtos';});
~~~

<h2 id="topico6">6. Agrupando rotas</h2>

- agrupar rotas através de um **prefixo** permite organizar melhor a parte de roteamento da aplicação.
- nesse caso, será utilizado pois nossa aplicação web terá uma parte visível para qualquer pessoa e uma parte restrita - ou seja, terá uma dimensão pública e outra privada (que dependerá de algum método de autenticação).
    - as rotas clientes, fornecedores e produtos serão agrupadas dentro de uma rota cujo prefixo será `/app`.
- em `web.php`:

~~~php
Route::prefix('/app') -> group(function() {
    Route::get('/clientes', function() {return 'Clientes';});
    Route::get('/fornecedores', function() {return 'Fornecedores';});
    Route::get('/produtos', function() {return 'Produtos';});
});
~~~

- executando o comando `php artisan route:list`, teremos as 3 novas rotas agrupadas da seguinte forma:

~~~
  GET|HEAD  app/clientes ............ routes/web.php:12
  GET|HEAD  app/fornecedores ........ routes/web.php:13
  GET|HEAD  app/produtos ............ routes/web.php:14
~~~

<h2 id="topico7">7. Nomeando rotas</h2>

- nesse caso, os nomes serão:
    - **rota principal (/)**: `site.index`.
    - **/sobrenos**: `site.sobrenos`.
    - **/contato**: `site.contato`.
    - **/login**: `site.login`.
    - **/clientes**: `app.clientes`.
    - **/fornecedores**: `app.fornecedores`.
    - **/produtos**: `app.produtos`.

~~~php
Route::get('/', [\App\Http\Controllers\PrincipalController::class, 'principal']) -> name('site.index');
Route::get('/sobre-nos', [\App\Http\Controllers\SobreNosController::class, 'sobreNos']) -> name('site.sobrenos');
Route::get('/contato', [\App\Http\Controllers\ContatoController::class,'contato']) -> name('site.contato');
Route::get('/login', function() {return 'Login';}) -> name('site.login');

Route::prefix('/app') -> group(function() {
    Route::get('/clientes', function() {return 'Clientes';}) -> name('app.clientes');
    Route::get('/fornecedores', function() {return 'Fornecedores';}) -> name('app.fornecedores');
    Route::get('/produtos', function() {return 'Produtos';}) -> name('app.produtos');
});
~~~

- os names podem ser utilizados através de uma **função helper** do framework laravel, ou seja, os names aplicados às rotas podem ser usados apenas dentro da lógica da aplicação (não podemos tentar acessar através do navegador "localhost:8000/app.clientes", por exemplo - a rota continua sendo app.clientes).
- aplicando os names em `principal.blade.php`, `sobre-nos.blade.php` e `contato.blade.php`:

~~~php
<ul>
    <li>
        <a href="{{ route('site.index')}}">Principal</a>
    </li>
    <li>
        <a href="{{ route('site.sobrenos')}}">Sobre nós</a>
    </li>
    <li>
        <a href="{{ route('site.contato')}}">Contato</a>
    </li>
</ul>
~~~

v

- recurso que permite direcionar o fluxo de navegação do usuário pela aplicação web.
- criaremos duas rotas adicionais apenas para fins de teste:rota1 e rota2.
- a ideia é que, caso caiamos na rota 2, sejamos encaminhados à rota1.

~~~php
Route::get('rota1', function() {
    echo 'Rota 1';
}) -> name ('site.rota1');

Route::get('rota2', function() {
    echo 'Rota 2';
}) -> name ('site.rota2');
~~~

- há algumas possibilidades diferentes de implementação de um redirecionamento de rota: podemos usar o **redirect do objeto route**, ou fazer **redirecionamentos dentro das funções de callback das rotas ou dentro dos Controllers**.
- pelo `redirect do objeto route`:

~~~php
/*
Route::get('rota2', function() {
    echo 'Rota 2';
}) -> name ('site.rota2');
 */

Route::redirect('rota2', 'rota1');
~~~

- `dentro da função de callback da rota` ou `dentro do Controller`, o que é mais comum:

~~~php
Route::get('rota2', function() {
    return redirect() -> route(site.rota1);
}) -> name ('site.rota2');
~~~

<h2 id="topico9">9. Rota de contingência (fallback)</h2>

- é uma rota de contingência, que será disponibilizada para o usuário caso a rota desejada por ele não seja localizada dentro do sistema de rotas da nossa aplicação.
- em `web.php`:

~~~php
Route::fallback(function(){
    echo 'A rota acessada não existe! <a href= "'.route('site.index').'">Clique aqui</a> para ir para a página inicial :)';
});
~~~
