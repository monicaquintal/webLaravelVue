<h2 id="topico1">1. Iniciando o projeto</h2>

```

composer create-project laravel/laravel meu-projeto
cd meu-projeto
```

<h2 id="topico2">2. Produtividade com Artisan Console</h2>

- artisan é a interface de linha de comando do framework Laravel, fornece auxílio para a realização de diversas tarefas repetitivas que são necessárias durante o desenvolvimento da aplicação.
- sempre que iniciamos um novo projeto, o artisan é automaticamente incluído.
- podemos também criar nossos próprios comandos para automatizar rotinas especializadas.
- fica localizado na raiz do projeto; para utilizá-lo, acessar a linha de comando `php artisan list`.
- alguns comandos importantes:

~~~
php artisan -V //mostra a versão do framework
php artisan serve // serve a aplicação através do servidor embutido do php
(podemos ainda passar parâmetro --port=9000 ou o número da porta desejada)
php artisan down // coloca nossa aplicação no modo de manutenção para realizar alguma operação que leve algum determinado tempo
php artisan up // volta para produção 
~~~

<h2 id="topico3">3. Rotas (Routes): Introdução</h2>

- **rotas** representam os caminhos existentes dentro da aplicação: ao acessar um recurso dentro da aplicação, atraves de uma URL, estamos, na prática, acessando uma rota.
- exemplos:

~~~
www.minhaaplicacao.com.br
www.minhaaplicacao.com.br/sobrenos
www.minhaaplicacao.com.br/contato
~~~

- no framework laravel, as rotas estão organizadas em arquivos separados: API, Channels, Console, Web são os arquivos na versão 7.
    - na versão 13, apenas `console.php` (serve para criação de comandos personalizados que serão executados a partir do artisan) e `web.php` (registra rotas que trabalharão processando páginas do lado do backend e servindo em resposta às requisições, permitindo uso de recursos como cookies e sessão).
    - o Laravel moderno passou a adotar uma estrutura mais enxuta e nem todas as rotas são criadas por padrão.

<h2 id="topico4">4. Super Gestão - Implementando as rotas principal, sobre-nos e contato</h2>

- nesse momento, trabalharemos com desenvolvimento web tradicional, aquele em que as views são renderizadas do lado do servidor; portanto, pra implementar as rotas,usaremos o arquivo `web.php`, contido dentro do diretório routes.
- a princípio, já existe no arquivo a `rota principal` (abaixo); qualquer acesso feito diretamente para a aplicação, quando é servida, é direcionada para essa rota, desde que não indiquemos nenhuma rota específica.

~~~php
<?php

use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('welcome'); //view padrão do laravel, para teste
});
~~~

- no projeto, executar o comando `php artisan serve`.
- remover a view e inserir o texto:

~~~php
<?php

use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return 'Olá, seja bem-vindo! :)'; 
});
~~~

- implementar as rotas sobre-nos e contato:

~~~php
<?php

use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return 'Bem-vindo!';
});

Route::get('/sobre-nos', function () {
    return 'Sobre nós';
});

Route::get('/contato', function () {
    return 'Contato';
});
~~~

- principais **verbos http** que podem ser utilizados para controlar as requisições que são feitas à aplicação: get, post, put, patch, delete e options.

<h2 id="topico5">5. Controladores (Controllers) - Introdução</h2>

- servem para agrupar a lógica do que deve ser feito em função da rota acessada por determinado cliente.
- ao fazermos uma requisição para o servidor (acessarmos uma rota), esta rota estará associada a um Controller e a uma action() (um controlador e uma ação definida dentro dele).
- a separação entre rotas e controladores permite separar melhor o código, definindo responsabilidades claras para cada parte do framework.
- embora rotas implementem funções de callback (as quais poderiam ser utilizadas para definir toda a lógica), o ideal é direcionarmos as requisições para os controladores, mantendo uma separação clara! 
    - essa separação é importante porque, no caso das rotas, focamos nos verbos http e middlewares (questões técnicas e de segurança da aplicação em si), enquanto os controladores têm foco na lógica necessária para atender aos requisitos do negócio (nos seus métodos e actions definimos o que a aplicação deve fazer em função do que foi determinado pela rota).

<h2 id="topico6">6. Super Gestão - Implementando os controladores principal, sobre-nos e contato</h2>

- controladores são classes: usar o padrão CamelCase, e terminar com a palavra Controller.
- para criar o Controller, utilizar o comando: `php artisan make:controller NomeDoController`.

~~~
php artisan make:controller PrincipalController
~~~

- ao executar esse comando, é criado um arquivo PrincipalController, no diretório app > Http > Controllers > PrincipalController.php, com a estrutura:

~~~php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class PrincipalController extends Controller
{
    //
}

~~~

- no `arquivo web.php`, comentar o trecho de código referente à página principal e inserir:

~~~php
/*
Route::get('/', function () {
    return 'Bem-vindo! :)';
});
*/

Route::get('/', '\App\Http\Controllers\PrincipalController@principal');

// ou 

Route::get('/', [\App\Http\Controllers\PrincipalController::class, 'principal']);

~~~

- e, no `arquivo PrincipalController.php`, inserir:

~~~php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class PrincipalController extends Controller
{
    public function principal() {
        return 'Olá, mundo! :)';
    }
}
~~~

- executar `php artisan serve`.
- agora que a aplicação funcionou normalmente, criar os demais Controllers:

~~~
php artisan make:controller SobreNosController
php artisan make:controller ContatoController
~~~

- em `ContatoController.php`, implementar:

~~~php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class ContatoController extends Controller
{
    public function contato() {
        echo 'Contato';
    }
}
~~~

- em `SobreNosController`, implementar:

~~~php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class SobreNosController extends Controller
{
    public function sobreNos() {
        echo 'Sobre nós';
    }
}
~~~

- e em `web.php`:

~~~php
Route::get('/sobre-nos', [\App\Http\Controllers\SobreNosController::class, 'sobreNos']);

Route::get('/', [\App\Http\Controllers\ContatoController::class,'contato']);
~~~

<h2 id="topico7">7. Visualizações (Views) - Introdução</h2>

- as Views são as visões que produzimos do lado do servidor e retornamos na sequência para o cliente que fez a requisição - esse é conhecido como o modelo tradicional de aplicações web.
- atualmente há uma técnica mais moderna de desenvolvimento web, baseada na separação de front e backend em aplicações distintas, conectadas via API.
- nesse curso serão trabalhadas as duas opções.

<h2 id="topico8">8. Super Gestão - Implementando as visualizações principal, sobre-nos e contato</h2>

- as views conterão todo o HTML das páginas que serão processadas do lado do backend: serão de fato as telas da aplicação.
- na raiz do projeto, acessar diretórios: resources > `views`.
    - criar dois subdiretórios: `site` e `app`.
    - em `site`, criar os arquivos `principal.blade.php`, `sobre-nos.blade.php` e `contato.blade.php`.
        - a instrução **.blade.php** é umn padrão do framework laravel, pois as views serão processadas por um recurso do laravel chamado blade.

- em `principal.blade.php`:

~~~php
<h3>Principal (view)</h3>
~~~

- em `contato.blade.php`:

~~~php
<h3>Contato (view)</h3>
~~~

- em `sobre-nos.blade.php`:

~~~php
<h3>Sobre nós (view)</h3>
~~~

- ajustar os Controllers:

~~~php
// PrincipalController.php
<?php
namespace App\Http\Controllers;
use Illuminate\Http\Request;
class PrincipalController extends Controller
{
    public function principal() {
        return view('site.principal');
    }
}

// SobreNosController.php
<?php
namespace App\Http\Controllers;
use Illuminate\Http\Request;
class SobreNosController extends Controller
{
    public function sobreNos() {
        return view('site.sobre-nos');
    }
}

// ContatoController.php
<?php
namespace App\Http\Controllers;
use Illuminate\Http\Request;
class ContatoController extends Controller
{
    public function contato() {
        return view('site.contato');
    }
}
~~~
